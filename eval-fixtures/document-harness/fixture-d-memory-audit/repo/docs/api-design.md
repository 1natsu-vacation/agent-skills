# API 設計

## レスポンス形式

レスポンスの JSON キーは snake_case を採用する。理由: 初期クライアントが Python 製で、チーム内の Python 規約と揃えるため。

例:

```json
{
  "user_id": "u_123",
  "created_at": "2025-11-01T00:00:00Z"
}
```

## エラーレスポンス

エラーは `code`（機械可読）と `message`（人間可読）の2フィールドで返す。HTTP ステータスと `code` を二重に持つのは、ステータスだけでは区別できない業務エラー（例: 送信先チャネル凍結）があるため。
