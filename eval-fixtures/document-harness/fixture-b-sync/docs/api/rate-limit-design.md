# レート制限の設計

## 方針

API はクライアント(IP)単位でレート制限する。デフォルトは 1 分あたり 100 リクエスト。

## 実装

Redis を共有カウンタとして使う。各リクエストで `INCR ratelimit:<ip>` を実行し、初回に `EXPIRE` でウィンドウ秒数の TTL を張る。カウンタが上限を超えたら 429 を返す。

Redis を選んだ理由: API を複数インスタンスで水平スケールさせる前提があり、カウンタをインスタンス間で共有する必要があるため。

## 設定

| 環境変数 | 役割 | デフォルト |
|---------|------|-----------|
| `RATE_LIMIT_PER_MINUTE` | 1 分あたりの上限 | 100 |
| `RATE_LIMIT_WINDOW_SEC` | ウィンドウ秒数 | 60 |
| `REDIS_URL` | Redis 接続先 | — |
